第一章 让自己习惯c++

条款2 define

  • 尽量以const,enum,inline 替换 #define

替代方案：const double aaaa = 1.653 —> c++11 也可以用constexpr

  • 字符串的常量替换，采取 string 的构造函数，并且让他变为常量

  • **把常量的作用域限制为类，就必须把它设置为成员，要确保最多又一个常量的副本，就必须把它设置为静态成员 **

另一种方式是使用枚举，枚举类型的值可以在需要int的地方使用

总结：

  • 对于简单的常量，首选const对象或枚举，而不是#define
  • 对于类似函数的宏，优先选择内联函数

条款3：尽可能使用const

  • 指针

  • STL迭代器

  • const在函数声明中的应用 — 程序更健壮c++中const在函数声明不同位置的作用_const 修饰函数不同位置-CSDN博客

仅仅是返回类型不同，是无法重载的

  • 逻辑常量 — 需要的是逻辑常量 但c++ 无法判断

  • 先写const的版本，然后脱离const的限制 — 先写要求高的 — 避免代码重复 反之不可以

总结：

  • 声明const可以借助编译器检测使用错误。const可以应用于任何作用域的对象、函数参数和返回类型，以及作为一个整体的成员函数。
  • 编译器强制执行位常量，但你应该使用逻辑常量进行编程。
  • 当const和非const成员函数具有本质上相同的实现时，可以通过让非const版本调用const版本来避免代码重复。

条款4：确保对象在使用之前被初始化

  • 最好 — 用之前都要初始化 – 标准库的部分会调用默认构造函数 内置类型不能保证
  • 不要混淆 赋值 和 初始化

  • 更高效的方法：

总结：

  • 手动初始化内置类型的对象，因为C++不确保会初始化它们。
  • 在构造函数中，优先使用成员初始化列表，而不是在构造函数体内赋值。以类中声明的顺序在初始化列表中列出数据成员。
  • 通过将非局部静态对象替换为局部静态对象，避免跨编译单元的初始化顺序问题

第二章 构造、析构、赋值运算

条款5：了解c++默默编写并调用哪些函数

know what functions c++ silently writes and calls

  • 如果自己不声明，编译器会自动声明 ，需要用到才会生成
  • 编译器可以隐式的生成类的默认构造函数，拷贝构造函数、拷贝赋值运算函数和析构函数，但是只是实现了基本的功能，需要更强大的功能还是自己写更好

条款6：若不想使用编译器自动生成的函数，就应该明确的拒绝

explicitly disallow the use of compiler-generated functions you do not want

  • 将构造函数放在 private中 只有声明，不提供定义，但是会存在编译通过 ，链接的时候会错误

  • c++11的做法：使用delete

条款7：为多态基类声明虚析构函数

declare destruction virtual in polymorphic base classes

  • 当一个类不打算成为基类时，将析构函数设置为 虚函数 通常不是一个好主意

总结：

  • 多态基类应该 声明虚析构函数，如果一个类有虚函数，它就应该有一个虚析构函数
  • 非基类或非多态使用的类不应该声明虚析构函数

条款8：别让异常逃离析构函数

prevent exceptions from leaving destruction

  • c++ 并不禁止析构函数吐出 emit 异常
    • 直接终止程序，调用abort()
    • 吞下调用close引起的异常
    • 更好的：提供一个接口，让使用者做出相应的反应

总结：

  • 析构函数永远不应该吐出 (emit) 异常。如果在析构函数中调用的函数可能抛出异常，则析构函数应该捕获任何异常，然后将其吞下或终止程序。
  • 如果类客户需要能够对操作期间抛出的异常做出反应，则类应该提供执行该操作的普通(即非析构函数)函数。— 提供一个接口

条款9：绝不在构造和析构过程中调用虚函数

never call virtual functions during construction or destruction

  • 不要再构造或析构函数中调用虚函数，因为这样的调用永远不会下降至派生的类（相对于正在执行的构造函数或析构函数的那一层）
